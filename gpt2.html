Text Generation using GPT

Open AI ChatGPT, Amazon Titan, Google Gemini, and Microsoft Co-pilot utilise NLP Algorithms to decipher text, allowing users to interact with devices using Natural Language. All these were possible because of the propelling force behind NLP, Artificial Intelligence (AI) using Neural Networks to mimic the human brain.

1. Introduction

This project focuses on fine-tuning OpenAI's GPT-2 model for text generation tasks. The model is trained on a dataset of news descriptions and evaluated using ROUGE and BLEU scores.

2. Data Preprocessing

2.1 Dataset

The dataset consists of a CSV file (bbc.csv) containing a description column with news text.

2.2 Preprocessing Steps

  • Tokenization of text using GPT-2 tokenizer
  • Truncation of long sequences to a maximum length of 1024 tokens
  • Conversion into PyTorch tensors

3. Model Training

3.1 Training Parameters

  • Model: GPT-2 (small variant)
  • Batch Size: 16
  • Epochs: 6
  • Learning Rate: 1e-5
  • Max Sequence Length: 500
  • Warmup Steps: 250
  • Optimizer: AdamW

3.2 Training Process

  • Data is loaded using PyTorchâ€™s DataLoader.
  • The model is fine-tuned using a supervised learning approach.
  • Loss is computed and backpropagated at each step.
  • Model checkpoints are saved at each epoch.

4. Text Generation

4.1 Generation Method

  • A seed text is provided to the model.
  • Text is generated using temperature sampling and top-p filtering.
  • The generated sequence is decoded back to text.

4.2 Example Output

Input Seed: "Breaking news: Scientists discover a new"

Generated Output: "Breaking news: Scientists discover a new species of deep-sea fish that glows in the dark. The discovery has amazed marine biologists worldwide."

6. Conclusion and Future Work

6.1 Conclusion

The fine-tuned GPT-2 model successfully generates coherent and contextually relevant text. The evaluation scores indicate that the model performs well in summarization and text generation tasks.

6.2 Future Work
